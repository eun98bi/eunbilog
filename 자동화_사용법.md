# eunbilog 자동 작성 사용법

baseball / affiliate / travel / ai-news 4개 카테고리는 터미널 명령이 아니라 **이 Claude
채팅에서 말로 지시**하면 된다. 실제 데이터(최신 뉴스, 쿠팡 베스트 상품, 마이리얼트립
인기 상품)는 외부 API로 가져오지만, **글 본문은 항상 Claude가 직접 쓴다.** Gemini API는
원본 사진이 없을 때 이미지를 만드는 용도로만 쓰인다.

---

## 사전 준비

`.env.local`에 아래 값이 채워져 있어야 한다 (이미 완료):

```
GEMINI_API_KEY=              # 이미지 생성용
COUPANG_ACCESS_KEY=
COUPANG_SECRET_KEY=
MRT_API_KEY=
NAVER_SEARCH_CLIENT_ID=
NAVER_SEARCH_CLIENT_SECRET=
```

**추가로 한 번만 설정할 것**: 원본 사진이 없는 글(주로 ai-news)에 Gemini가 이미지를
만들어 올릴 공개 저장 공간이 필요하다. Supabase 대시보드 → **Storage** → New bucket →
이름 `post-images`, **Public** 체크 → 생성. (schema.sql 실행과 동급의 1회 수동 설정)

---

## 사용 예시 (채팅에 이렇게 말하면 됨)

```
baseball 자동으로 써줘
baseball 한화 이글스 KIA 타이거즈만 자동으로 써줘
affiliate 자동으로 3개 써줘
travel 오사카 상품으로 자동으로 3개 써줘
ai-news 최신 소식으로 자동으로 써줘
```

- 검토만 하고 바로 게시하고 싶지 않으면 "초안으로 저장해줘"라고 붙이면 된다
  (`published:false`로 저장, `/admin`에서 확인 후 게시로 전환)
- 여러 건(예: "구단 10개 다") 요청하면 Claude가 항목별로 병렬 처리할 수도 있다

## 뒤에서 일어나는 일

1. Claude가 해당 데이터(뉴스 헤드라인 / 쿠팡 베스트 상품 / 마이리얼트립 인기 상품)를
   실제 API로 조회
2. 같은 카테고리 최근 글을 확인해 겹치는 주제를 피함
3. 조회된 사실만 근거로 Claude가 직접 제목/본문/태그/요약을 씀 (baseball은 뉴스에 없는
   스코어·수치를 지어내지 않음)
4. 이미지: 뉴스 대표사진/상품사진처럼 원본이 있으면 그대로 사용, 없으면(주로 ai-news)
   그때만 Gemini로 하나 생성해서 사용
5. affiliate/travel은 제휴 disclosure 문구와 실제 구매/예약 링크를 자동으로 포함
6. 완성된 글을 Supabase에 바로 게시 (또는 초안 저장)

게시된 글은 최대 60초 이내에 `eunbilog.com/blog/[category]/...`에 반영된다.

---

## 대상이 아닌 카테고리

gov-info, app-dev, side-hustle, tooltoolz, 에이블리 제휴는 이 자동 흐름 대상이 아니다.
gov-info는 정확성 요구가 높아 사람이 대화로 확인하며 쓰는 기존 방식이 더 안전하고,
나머지는 애초에 경험 기반 글이라 자동화 대상이 아니다. 이 카테고리들은 기존처럼
"[카테고리] [주제]로 써줘" 형식으로 대화하면 된다.

## 주의

- 쿠팡/마이리얼트립/네이버 API는 실제 크레덴셜과 쿼터를 사용한다. Claude는 요청받은
  횟수만큼만 실행하고 임의로 반복 실행하지 않는다.
- 자동 작성 글도 다른 글과 동일하게 `/admin`에서 확인·수정·삭제할 수 있다.
